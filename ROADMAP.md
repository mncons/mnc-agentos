# Roadmap

Este roadmap es **direccional**: describe los problemas que la plataforma se propone abordar, no un compromiso de fechas ni de alcance. Sirve para mostrar la clase de dominios donde el [patrón vertical](docs/architecture/vertical-pattern.md) se pone a prueba, no para anunciar productos.

Cada vertical se describe por el **problema que resuelve**, no por su marca. El conocimiento de dominio concreto —constituciones, corpus, rúbricas— vive fuera de este repositorio por diseño.

## Verticales en consideración

- **Asistente por voz para población con baja literacidad digital.** Llevar la interacción con un agente a personas para quienes la interfaz de texto es una barrera, priorizando voz y lenguaje llano.
- **Monitoreo de convocatorias de financiación.** Vigilar de forma continua la apertura de oportunidades de financiación y señalar las relevantes según criterios del usuario, reduciendo el costo de no enterarse a tiempo.
- **Optimización logística con principios de redes biológicas** *(línea de investigación)*. Explorar heurísticas inspiradas en sistemas biológicos auto-organizados para problemas de ruteo y asignación. Es investigación, no un producto comprometido.

## Cómo leer este roadmap

La prioridad real entre estos frentes responde a la disponibilidad de corpus, la madurez del problema y la validación temprana descrita en [evaluation.md](docs/architecture/evaluation.md). Una línea acá significa "problema que vale la pena abordar con este patrón", no "esto se entrega en el trimestre X".
